# Medaka stdlib

## `runtime.mdk` — built-in extern catalog

`runtime.mdk` is the authoritative source of type signatures for all built-in
primitives. It is written in Medaka `extern` syntax and embedded into the
compiler at build time via `lib/stdlib_content.ml` (generated by `gen/embed.ml`).

### Adding a new primitive

1. Add an `extern` line to `runtime.mdk` with the correct type signature.
   Use `<IO>`, `<Mut>`, `<Panic>`, etc. on the return type if the operation
   has side effects — the effect checker reads these annotations automatically.

2. Add a matching OCaml implementation to the `primitives` list in
   `lib/eval.ml`. The startup completeness assertion will fail at runtime if
   you forget this step.

3. Rebuild: `dune build`. The dune rule regenerates `lib/stdlib_content.ml`
   from `runtime.mdk`, so the new extern is immediately visible to the type
   checker and evaluator.

### Convention

- Pure functions: no effect annotation (`extern foo : a -> b`).
- IO operations: `<IO>` on the return type.
- Mutations via `set_ref`: `<Mut>`.
- Unrecoverable exits: `<Panic>`.
- Type variables in extern signatures are implicitly universally quantified.
