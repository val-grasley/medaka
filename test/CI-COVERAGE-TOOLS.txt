# CI-COVERAGE-TOOLS.txt — scripts in the repo that are TOOLS, not gates.
#
# diff_compiler_ci_shard_coverage.sh enumerates EVERY *.sh IN THE REPO and demands that
# each one be either:
#   (a) run in CI (matched by a shard `pattern:`, or named literally by a job step), or
#   (b) listed in CI-COVERAGE-EXCEPTIONS.txt with a reason (a LEDGER — printed every run), or
#   (c) listed HERE, because it is not a gate at all.
#
# WHY THE GATE ENUMERATES EVERYTHING NOW. It used to enumerate only the globs it already
# knew about (`test/diff_compiler_*.sh`, later plus wasm/bootstrap). So it certified
# coverage while a dozen REAL GATES — cross_project_*, diff_native_cli, the effect_*_domain
# family, selfcompile_emit, manifest_emit — had never run in CI at all. A completeness check
# that picks its own scope will always certify itself complete. The only way out is to make
# the scope EVERYTHING and force every script to be classified exactly once.
#
# AND THEN IT DID IT AGAIN (fixed 2026-07-13). "EVERYTHING" meant `test/*.sh` +
# `test/wasm/*.sh` — a curated pair of globs, not an enumeration. So 38 scripts sat outside
# the gate's scope, including 24 real gates: the 22 `sqlite/test/*_oracle.sh` differential
# gates (against the real sqlite3 CLI), `playground/e2e/run.sh`, and — inside test/ itself,
# one directory down — `test/native_fixtures/run.sh`, which was RED and had been for an
# unknown length of time. TWO OF THE ENTRIES IN THIS FILE WERE ALSO LIES: `build_cmd` and
# `build_wasm_cmd` were filed here as "helpers" while being, by their own headers, real
# end-to-end gates over the shipping `medaka build` CLI (14 and 4 assertions, both ending
# in `[ "$fail" -eq 0 ]`). They are now in a shard / the ledger. The scope is now literally
# every `.sh` in the tree.
#
# A script in this file makes a claim: "running me proves nothing about the compiler."
# If that is false, it belongs in a shard, not here. That claim was FALSE for two of the
# entries this file used to carry. Do not add a line here to make a red gate go away —
# that is what CI-COVERAGE-EXCEPTIONS.txt is for, and it demands a reason and an owner.
#
# KEYS ARE REPO-RELATIVE PATHS (minus `.sh`), not basenames — basenames collide across
# roots (test/native_fixtures/run.sh vs playground/e2e/run.sh both stem to "run"), and a
# ledger keyed on a colliding name classifies the wrong file.
#
# Format:  <repo-relative-path-without-.sh>  <what it is>

# ── build / bootstrap ──

test/bootstrap_from_seed       cold-bootstraps from the seed. Run by the `seed-health` CI job by name.
test/build_oracles             builds test/bin probe binaries. The gates' dependency, not a gate.
test/wasm/build_wasm_oracle    builds the wasm probe binaries.
test/wasm/assemble_check_main  probe assembly helper.
test/refresh_seed              re-mints compiler/seed/emitter.ll.gz. A maintenance action.

# ── harness / driver ──
test/run_gates                 THE gate runner. Running it IS running the gates.
test/preflight                 the agent inner loop — derives a gate subset from `git diff`.
test/capture_goldens           (re)captures goldens. A blessing action, not a check.

# ── golden capture ──
# CHECK=1 turns this one into a comparison, but its DEFAULT action is to WRITE the goldens
# and nothing invokes it with CHECK=1. It is a blessing action, exactly like capture_goldens.
# (If the error-quality corpus is ever wired into CI it goes in a shard as CHECK=1 — and
# then this line comes out.)
test/error_quality_fixtures/capture   captures the error-quality baseline .out goldens.

# ── measurement (no pass/fail verdict) ──
test/bench                     perf harness. Reports numbers; asserts nothing.
test/profile_compiler          per-stage time/allocation profiler.
test/tmc_census                emits the TMC census that diff_compiler_tmc_parity CONSUMES.
test/argstamp_parity_probe     an investigation probe from the argstamp arc.

# ── sourced libraries (never executed; `sh <file>` is a no-op) ──

                               nothing observable, so there is nothing for a CI shard to assert.
