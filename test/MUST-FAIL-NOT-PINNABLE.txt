# MUST-FAIL-NOT-PINNABLE.txt — issues deliberately NOT pinned by the must-fail suite,
# each with a reason. Read by test/must_fail_census.sh (#569).
#
# The census (test/must_fail_census.sh) reports open+verified issues that have no fixture
# in test/must_fail_fixtures/. Large parts of the tracker are legitimately UN-pinnable:
# gate/CI defects, doc defects, perf work, and code health have no `main.mdk` to pin and
# no exact observable for `diff_compiler_must_fail.sh` to grade. Listing one here says:
# "a human looked at this and judged it out of shape for THAT HARNESS" — it is not a
# statement that the bug is unimportant, and it is not a wontfix.
#
# ── A LEDGER, NOT A LABEL, AND NOT A SKIP-LIST ────────────────────────────────
#
# This is a file rather than a `not-pinnable` GitHub label ON PURPOSE. A label has no
# REASON FIELD and needs no review: anyone can silence the census with one click, with no
# PR, no reviewer, and no rationale anybody can audit later. That is a skip-list, and
# **a skip-list cannot notice when it stops earning its place, so it rots** (that is how
# test/ported/ died). An entry here costs a PR and a sentence a reviewer reads.
#
# ── SELF-DRAINING IN BOTH DIRECTIONS, EACH HALF CHECKED WHERE IT CAN BE ────────
#
# `.claude/ORCHESTRATING.md`: an exceptions ledger must FAIL when an entry stops earning
# its place — BOTH when the excused thing came back AND when nothing cites it anymore.
# Half a ratchet is a skip-list. So:
#
#   * test/diff_compiler_must_fail.sh (the GATE — tree-only, no network) FAILS when an
#     entry HAS A FIXTURE after all. The corpus contradicts the exemption: someone found
#     a way to pin it, and the entry is now a lie.
#   * test/must_fail_census.sh (NIGHTLY — has the GitHub API) reports an entry whose
#     issue is CLOSED. The exemption has no subject left; remove the line.
#
# Neither half can check the other's: the gate has no network (it runs on every dev box
# via `make gates`), and only the API knows whether an issue is still open.
#
# Format (one entry per line; blank lines and '#' comments ignored):
#   <issue-number>  <reason — MANDATORY, and it must say what SHAPE the problem is>
#
# A reason that does not survive being read aloud to a reviewer is not a reason. "Hard"
# and "not worth it" are not shapes. Name the thing the harness cannot express.
