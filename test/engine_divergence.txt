# test/engine_divergence.txt — the KNOWN-FAILURE LEDGER for diff_compiler_engines.sh.
#
# rustc's tests/crashes model: every line asserts the CURRENT (wrong or degraded)
# behaviour of a fixture, so that (a) nothing regresses further, and (b) AN
# ACCIDENTAL FIX IS DETECTED — the gate FAILS if a line here starts passing, and
# tells you to promote it by deleting the line.  A plain skip-list cannot do that,
# and a silent skip is how this suite already lies to itself (TESTING-DESIGN §2.3).
#
# Format:   <key>  <signature>  <category>  <reason>
#
#   key        <corpus>/<fixture basename>, over the four emitter corpora:
#                llvm/   test/llvm_fixtures/        untyped, prelude-free
#                llvmT/  test/llvm_fixtures_typed/  its typed sibling
#                wasm/   test/wasm/fixtures/        untyped, prelude-free
#                wasmT/  test/wasm/fixtures_typed/  its typed sibling
#              The corpus prefix is load-bearing: 18 basenames exist in more than one
#              corpus as DIFFERENT files (arr_get, str_lit, fn_gcd, adt_option, …).
#
#   signature  <en>:<nw>:<ew>:<eval>:<native>:<wasm>
#                en / nw / ew  the three PAIRWISE verdicts — eval-vs-native,
#                              native-vs-wasm, eval-vs-wasm — each  eq | ne | na
#                eval/native/wasm  whether that arm could run at all —  ran | na
#              All three pairs are recorded rather than one collapsed "agree" flag,
#              so that when an arm is unavailable (no wasm-tools, say) the gate can
#              still soundly enforce the pairs that DO NOT depend on it, instead of
#              dropping the whole fixture.
#
#   category   one of (see test/ENGINE-DIVERGENCE.md for the full write-up):
#                eval:rng-hash-lcg          eval:extern-not-implemented
#                eval:no-tco                eval:unsupported-node
#                eval:intended-abort        native:prelude-collision
#                native:autoprint-ambiguous wasm:emitter-gap
#                wasm:codegen-bug           emitter:shared-eager-init
#
#              `emitter:shared-eager-init` (added 2026-07-16 for #553) is the first
#              category naming a defect in a SHARED emitter primitive rather than in one
#              engine.  It was added rather than bending an existing label because every
#              existing one would have been a FALSE claim: the `wasm:*` categories are
#              wrong (native is broken too — it SIGSEGVs on the boxed shape and silently
#              prints a zero on the unboxed one), and there is no `native:*` codegen
#              category at all.  The taxonomy simply had no cell for "one shared helper
#              (`eagerVars`), both backends wrong", and this file's own rule — THE
#              CATEGORY IS A CLAIM — forbids writing the nearest label instead.
#
#   reason     the specific bug, one line.  NEVER leave it as TODO.
#
# ⚠️ THE CATEGORY IS A CLAIM. VERIFY IT — a wrong one launders a bug behind a plausible
# label, and it will be repeated as fact by everyone who reads this file. It HAS been:
# four `wasm:emitter-gap` rows for the w7_array_* fixtures asserted a wasm ref-mode gap
# that DID NOT EXIST, and an orchestrator briefed an agent on that wrong story as if it
# were established (rows deleted 2026-07-13). On 2026-07-14 TWELVE MORE `wasm:emitter-gap`
# / `wasm:codegen-bug` rows dissolved the same way — char_lit, char_unicode, uni_to_lower,
# uni_to_upper, num_pi, num_e, abort_exit, abort_exit_nonzero, where_local_rec,
# arr_set_unit, debug_charlit, debug_strlit, fn_bool_return, fn_float_chain. Those were
# artifacts of the gate's own wasm arm, and they sat here for weeks wearing a backend's
# name.
#
# ⚠️ BUT THE INVERSE IS ALSO A CLAIM, AND IT WAS ALSO WRONG. An earlier revision of this
# paragraph listed num_int_min, num_int_max and where_sibling_ref among the dissolved rows
# and asserted "NONE of them were wasm bugs". That was FALSE. Their OLD row text was
# wrong (it described a prelude-free-probe artifact), but once the gate's wasm arm moved
# onto the shipping `medaka build --target wasm` all three failed for REAL, were re-filed
# as genuine `wasm:codegen-bug`s, and were then genuinely FIXED in wasm_emit.mdk

# gate artifact" is exactly as much of a claim as "it is a backend bug" — DEBUNKING a row
# needs the same proof as filing one. Read the emitted `.wat` (`medaka build --target wasm
# --keep-ir`) before you write either sentence.
#
# ── THE DISCRIMINATING TEST (rewritten 2026-07-14) ────────────────────────────────────
# It used to say: run the fixture through the two prelude-free probe entries
# (test/bin/{llvm,wasm}_emit_main) and see whether ONLY wasm fails. THAT ADVICE IS NOW
# OBSOLETE, AND IT IS WHAT PRODUCED THE 12 BAD ROWS ABOVE. Both probes run a compiler
# NOBODY SHIPS: parse → desugar → annotate → lower → emit, with NO TYPECHECK and NO
# PRELUDE. On that path `println` is unbound, `pi` is unbound, and `.[i]` (the `Index`
# interface method) cannot resolve — so the probe rejects correct programs, and it says so
# in words that read like a backend defect:
#
#     "wasm_emit gap — ref-mode: unbound variable 'index'"
#
# There is no gap in that sentence. There is a missing prelude. Do not take the emitter's
# own wording as the category.
#
# ⚠️ T-22 (2026-07-14) fixed this AT THE SOURCE: `wasm_emit.mdk`'s unbound-name lookup
# miss no longer routes through the generic `gap` wrapper (which unconditionally
# prepended "wasm_emit gap — " — a category CLAIM the emitter cannot verify) — it now
# panics with the bare OBSERVATION, e.g. `unbound variable 'index' (not a local, global
# value, constructor, or known function) [in <binding>]`. Every row below that still
# quotes the OLD "wasm_emit gap — ref-mode: unbound variable …" text as its verbatim
# `reason` has been mechanically updated to the new wording — those rows are STILL
# ledgered (the underlying unbound-name failures are real, on the shipping CLI, not probe
# artifacts), only the quoted panic text changed. See compiler/ERROR-QUALITY.md
# ("report what you observed, not what you concluded") and AGENTS.md T-22.
#
# Since 2026-07-14 all three arms of the gate ARE the shipping CLI, so the discriminating
# test is simply to run the two shipping backends and compare:
#
#     ./medaka build              --allow-internal <fixture> -o /tmp/n && /tmp/n
#     ./medaka build --target wasm --allow-internal <fixture> -o /tmp/w.wasm \
#         && node test/wasm/run.js /tmp/w.wasm
#
#   * BOTH fail the same way  -> NOT a backend gap. It is a front-end fact about the
#     fixture (e.g. it redefines a prelude interface, so the real prelude collides with
#     it). Use `native:prelude-collision`, not `wasm:*`.
#   * ONLY wasm fails         -> a genuine wasm gap, on the path users actually run.
