# fuzz_allowlist.txt — documented-open gaps the differential fuzzer must not
# treat as new findings.  KEEP MINIMAL: only gaps that are already filed in
# PLAN.md / selfhost/CONSTRUCT-COVERAGE.md, each with its citation.
#
# Format: one extended-regexp per line (matched by `grep -E` against a failing
# program's SOURCE and/or against the divergence output).  Lines starting with
# `#` are comments.  A failing seed whose source OR diff matches any pattern is
# logged as KNOWN-GAP, not a new finding.
#
# The Stage-0/1 MVP generator deliberately stays inside constructs that BOTH
# backends agree on, so this list is nearly empty by design.  The one entry
# below documents the named-field-record divergence the generator AVOIDS
# emitting (it is here so a future generator that re-enables records, or a
# hand-added corpus program, is correctly classified rather than mis-reported).
#
# NOTE on operator dispatch: Gap G (==/!=/</>/<=/>= dispatching to user/derived
# Eq/Ord) is CLOSED on synced main for the interpreter + selfhost tree-walker
# (PLAN.md Phase 151 / MEMORY project_phase151_gapg_operator_dispatch).  The
# fuzzer's Tier-B target IS the tree-walker, so derived-ADT comparison is
# EXPECTED to agree — NOT allowlisted.  (Only native-`<` is deferred, and the
# native tier is out of MVP scope.)
#
# NOTE on named-field `data` deriving: derived Eq/Ord/Debug on a named-field
# `data` variant (`data T = C { f : .. } deriving (..)`) is now CLOSED on the
# selfhost tree-walker too — the tree-walker's ERecordCreate produces a
# positional VCon for registered named-field constructors (mirroring lib/eval.ml
# ctor_field_order), so the derived positional `PCon` impl bodies match.  This
# was previously allowlisted as a known divergence; it is now fixed and EXPECTED
# to agree, so the entry has been removed.
#
# ============================ TIER-C (NATIVE) =============================
# Patterns below classify a `medaka build` (native LLVM backend) failure as a
# DOCUMENTED-OPEN native gap rather than a new finding.  They are matched ONLY
# against the captured build/run error text (the harness passes the native
# stderr as the `extra` arg), never against ordinary program source, so they
# cannot accidentally bury a Tier-A/Tier-B finding.
#
# IMPORTANT — these are DEFENCE-IN-DEPTH.  The type-directed Tier-C generator emits
# NO open-native-gap construct: `debug`/`==`/`compare` on a tuple receiver now build
# native==oracle (the arity-distinguished tuple-head fix closed former Gap C1/C5), so
# tuples are no longer suppressed; and the generator never compares a tuple with
# `<`/`>` (the one deferred parametric-default-`<` gap is unreachable).  So in normal
# Tier-C operation NOTHING here should fire; an entry firing means the generator grew
# a new construct reaching an open gap, classified as the already-filed one cited.
# Keep MINIMAL: each line is one exact documented panic.
#
# Gap C (RESIDUAL) — arg-tag dispatch on a constructorless PRIMITIVE / polymorphic
# receiver (the tuple case is CLOSED; primitive/polymorphic still open).
# selfhost/CONSTRUCT-COVERAGE.md §"Gap C"; selfhost/EMITTER-GAPS.md row 2.  Exact
# emitter panic:
arg-tag dispatch on impl type that owns no constructors
no impl groups
#
# max/min over a generic Ord receiver — selfhost/EMITTER-GAPS.md §"Open emitter
# CAPABILITY gaps" + TOTAL-row residual (2 events).  In `medaka build` the DCE
# pass (selfhost/dce.mdk) prunes the unreachable core.mdk `maximum`/`minimum`
# bodies, so this should NOT fire from `medaka build`; the entry is here only so
# a program that calls `max`/`min` itself (keeping them reachable) is classified
# rather than mis-reported.  Emitter signature mentions the unresolved compare:
no impl of method 'max'
no impl of method 'min'
#
# Two-level nested dicts (`Box (List (List Int))`) — SIGSEGV, emitter value-rep
# limit.  selfhost/EMITTER-GAPS.md / PLAN.md Stage 3.  The type-directed generator
# never nests containers inside ADT fields (variant fields are scalar-only,
# fuzz_gen.ml `field_tys`), so this is unreachable today; entry is documentation.
# (No reliable text signature — a SIGSEGV is caught by the harness as a build/run
# crash and reported as a NEW finding deliberately, so it is NOT pattern-matched
# here: a segfault we did not anticipate MUST surface, not be buried.)
