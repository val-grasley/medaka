# test/DOC-LINK-EXCEPTIONS.txt — checked-in ledger of every KNOWN dead doc
# reference. Read test/check_doc_links.sh first.
#
# This is a RATCHET, not a skip-list. Every line must ACTIVELY EARN ITS PLACE
# on every run: the path is still dead AND something still cites it. When
# either half stops being true, the gate FAILS and the line must be deleted.
# test/check_doc_links.sh enforces all four ways a line can stop earning it:
#
#   1. STALE REF   — the excused path NOW EXISTS. The reference is no longer
#                    dead; leaving the line in would hide a FUTURE regression
#                    at that same path forever.
#   2. ORPHAN REF  — NO DOCUMENT CITES that path anymore. This is the other
#                    half of "no longer needed", and it is the half a naive
#                    skip-list misses. It has already bitten: a docs-cleanup PR
#                    fixed the add-language-feature skill by REMOVING its
#                    citation of test/diff_compiler_roundtrip.sh. The file still
#                    did not exist, so check 1 stayed silent — yet this ledger
#                    still carried a line whose reason said "cited by the live
#                    add-language-feature skill". That reason had become a LIE,
#                    and nothing would ever have reported it. As the docs-cleanup
#                    work drains these paths, EVERY drained entry would rot into
#                    a silent orphan and most of this file would become fiction.
#   3. STALE FILE  — the excused doc no longer exists.
#   4. ORPHAN FILE — the excused doc no longer has a SINGLE dead reference, so
#                    the blanket excuse is dead weight — and would silently
#                    excuse any FUTURE rot in that file.
#
# A ledger nobody re-checks is how test/ported/ and diff_compiler_lint_multi sat
# silently skipped-and-failing for months; this file cannot do that, because the
# gate re-verifies every line on every run.
#
# NOTE: a gate that has to excuse its OWN false positives is a gate with a
# parsing bug. Link targets containing a raw space (DICT-SEMANTICS.md math
# notation) and ellipsis-elided paths (`compiler/.../resolve.mdk`) are therefore
# dropped at EXTRACTION in check_doc_links.sh — not carried as exceptions here.
#
# Format (TAB-separated; blank lines and '#' comments ignored):
#   KIND<TAB>pattern<TAB>reason
#
#   KIND ∈ FILE | REF
#     FILE  pattern is a citing-file shell glob (`case` syntax, e.g.
#           "archive/*" or an exact filename), matched against the file's
#           git-relative path. EVERY dead reference found while scanning a
#           matching file is excused. Use for a doc that is, in its
#           entirety, a point-in-time record — a completed audit/plan, a
#           dated log, a bootstrap narrative — where the correct fix is
#           "leave it alone", not "update the paths": rewriting a closed
#           record's paths to match today's tree would make it describe a
#           tree that never existed at the time the record was written.
#     REF   pattern is the exact resolved target path (root-relative, as
#           this gate normalizes it), matched exactly. Every citation of
#           that dead target, in ANY file, is excused. Use for a one-off
#           stale reference inside an otherwise-LIVING doc — a doc that is
#           still current, actively read, and whose OTHER links must stay
#           held to account.
#
# Reason convention: a reason starting with "TODO(docs-cleanup):" marks a
# REAL bug — a stale path from the compiler subfolder restructure (or a
# comparable rename/removal) sitting in a doc that is still current and
# should say the right thing. Any other reason marks genuine, permanent
# history.
#
