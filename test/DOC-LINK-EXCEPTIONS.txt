# test/DOC-LINK-EXCEPTIONS.txt — checked-in ledger of every KNOWN dead doc
# reference. Read test/check_doc_links.sh first.
#
# This is a RATCHET, not a skip-list: test/check_doc_links.sh FAILS if an
# entry here refers to a path that NOW EXISTS (REF) or a citing file that NO
# LONGER EXISTS (FILE, non-glob) — the maintainer must delete that line. A
# ledger nobody re-checks is how test/ported/ and diff_compiler_lint_multi
# sat silently skipped-and-failing for months; this file cannot do that,
# because the gate re-verifies every line on every run.
#
# Format (TAB-separated; blank lines and '#' comments ignored):
#   KIND<TAB>pattern<TAB>reason
#
#   KIND ∈ FILE | REF
#     FILE  pattern is a citing-file shell glob (`case` syntax, e.g.
#           "archive/*" or an exact filename), matched against the file's
#           git-relative path. EVERY dead reference found while scanning a
#           matching file is excused. Use for a doc that is, in its
#           entirety, a point-in-time record — a completed audit/plan, a
#           dated log, a bootstrap narrative — where the correct fix is
#           "leave it alone", not "update the paths": rewriting a closed
#           record's paths to match today's tree would make it describe a
#           tree that never existed at the time the record was written.
#     REF   pattern is the exact resolved target path (root-relative, as
#           this gate normalizes it), matched exactly. Every citation of
#           that dead target, in ANY file, is excused. Use for a one-off
#           stale reference inside an otherwise-LIVING doc — a doc that is
#           still current, actively read, and whose OTHER links must stay
#           held to account.
#
# Reason convention: a reason starting with "TODO(docs-cleanup):" marks a
# REAL bug — a stale path from the compiler subfolder restructure (or a
# comparable rename/removal) sitting in a doc that is still current and
# should say the right thing. Any other reason marks genuine, permanent
# history (or a gate-extraction false positive that isn't a doc bug at all).
#
# ── Split, as of the seeding pass (2026-07-13) ──────────────────────────────
#   13 FILE exceptions — all point-in-time records (closed audits/plans,
#       dated logs, the bootstrap narrative, the two archive/ docs, and
#       PLAN-ARCHIVE.md). Legitimate history; not counted as debt.
#    3 REF exceptions  — gate-extraction false positives (a placeholder
#       filename in an illustrative example, and math notation in
#       DICT-SEMANTICS.md that happens to look like `](...)`). Not doc bugs.
#   30 REF exceptions  — TODO(docs-cleanup): real stale paths in LIVING docs.
#       This is the punch list: mostly the compiler/*.mdk -> subfolder move
#       (compiler/typecheck.mdk -> compiler/types/typecheck.mdk etc.), plus a
#       few from the OCaml removal (lib/eval.ml) and the retired bytecode
