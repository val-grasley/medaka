# test/AGENT-DOC-SYMBOL-EXCEPTIONS.txt — checked-in ledger of every KNOWN
# backticked symbol-shaped token in the agent-facing docs that resolves
# nowhere in compiler/*.mdk, stdlib/*.mdk, or runtime/*.c, but is NOT a
# fabrication. Read test/check_agent_doc_symbols.sh first.
#
# This is a RATCHET, not a skip-list. Every line must ACTIVELY EARN ITS PLACE
# on every run: the token is still unresolved AND something still cites it.
# When either half stops being true, the gate FAILS and the line must be
# deleted. test/check_agent_doc_symbols.sh enforces all four ways a line can
# stop earning it:
#
#   1. STALE SYM   — the excused token NOW RESOLVES in source. The claim is
#                    no longer dead; leaving the line in would hide a FUTURE
#                    regression at that same name forever (e.g. if a symbol
#                    genuinely gets renamed BACK to a name sitting in here).
#   2. ORPHAN SYM  — NO document cites that token anymore. This is the other
#                    half of "no longer needed" — the half a naive skip-list
#                    misses. See test/DOC-LINK-EXCEPTIONS.txt's header for the
#                    exact historical incident this mirrors (a drained
#                    citation left an orphaned, now-fictional reason string).
#   3. STALE FILE  — the excused doc no longer exists.
#   4. ORPHAN FILE — the excused doc no longer has a SINGLE dead symbol claim,
#                    so the blanket excuse is dead weight — and would
#                    silently excuse any FUTURE fabrication in that file.
#
# Format (TAB-separated; blank lines and '#' comments ignored):
#   KIND<TAB>pattern<TAB>reason
#
#   KIND ∈ SYM | FILE
#     SYM   pattern is the EXACT token (case-sensitive), excused everywhere
#           it's cited across the whole doc corpus. Use for a one-off token
#           that is genuinely not a source-code citation, in an otherwise-
#           LIVING doc whose other claims must stay held to account.
#     FILE  pattern is a citing-file shell glob (`case` syntax — an exact
#           filename has no metacharacters and is matched literally), and
#           excuses EVERY dead symbol claim found while scanning a matching
#           file. Use ONLY when a doc's STATED PURPOSE is to name symbols
#           that do not exist yet — see the STDLIB.md entry below, whose own
#           header says "Owns: missing stdlib/support functions." That is a
#           structurally different genre from the playbooks this gate exists
#           to guard: a backlog says "build this", a playbook says "this is
#           real, use it" — only the second kind can silently mislead an
#           agent into calling a function that was never there.
#
