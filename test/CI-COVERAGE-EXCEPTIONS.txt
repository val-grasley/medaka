# CI-COVERAGE-EXCEPTIONS.txt — gates deliberately NOT run in CI, each with a reason.
#
# diff_compiler_ci_shard_coverage.sh requires EVERY gate script to be either
#   (a) matched by a shard `pattern:` in .github/workflows/ci.yml, or
#   (b) named literally by some step in ANY workflow under .github/workflows/
#       (as of T-4 this includes nightly.yml, a scheduled home for gates
#       deliberately unsuited to the PR path — see the entry below), or
#   (c) listed HERE, with a reason.
#
# This file is a LEDGER, not a skip-list. The distinction is the whole point:
# a skip-list rots silently, because nothing ever revisits it. A ledger entry is
# PRINTED BY THE GATE ON EVERY RUN, names the reason, and names the task that
# retires it — so it is impossible to forget and expensive to leave lying around.
#
# Adding a line here is a deliberate, reviewable act. "It's broken so I excluded
# it" is only acceptable with a task number next to it and an owner fixing it.
#
# KEYS ARE REPO-RELATIVE PATHS (minus `.sh`), not basenames. Gates do not all live
# under test/ — the coverage gate now enumerates every `.sh` in the repo — and
# basenames collide across roots (test/native_fixtures/run.sh vs
# playground/e2e/run.sh both stem to "run"). A ledger keyed on a colliding name is a
# ledger that excuses the wrong file.
#
# Format:  <repo-relative-path-without-.sh>  <reason>

# ── wasm backend (5 gates) — NOW IN CI (T25, 2026-07-15) ──────────────────────
# The five WasmGC gates — test/wasm/diff_wasm, diff_wasm_typed, diff_wasm_modules,
# diff_sqlite, and test/build_wasm_cmd — previously ran in NO CI job: a stock
# ubuntu-latest runner has neither wasm-tools nor a WasmGC-capable node>=22, and the
# gates SKIP when the toolchain is absent. That blind spot is how two silent wasm
# miscompiles (a `round` divergence, intToFloat #199) reached main.
#
# T25 stood up the `wasm` job in .github/workflows/ci.yml. It installs node 24 +
# wasm-tools, builds ./medaka + the wasm emitter oracle, and runs all five gates NAMED
# by their literal repo-relative path — which is what makes
# diff_compiler_ci_shard_coverage.sh classify them as covered (the same mechanism the
# soundness job uses for the fixpoint/typecheck gates). So they need NO row here.
#
# All five verified GREEN on the shipping CLI when T25 landed (149/0, 9/0, 36/0, 11/0,
# 4/0) — INCLUDING the two rows this ledger had recorded as RED, which were STALE, the
# #71 probe-artifact pattern:
#   * diff_wasm's "4 ref-mode failures" — actually 149/0, every IR-shape assert green.
#   * diff_wasm_typed's disp_hof_shadows_method — a REGRESSION FIXTURE for an
#     already-fixed miscompile; `medaka build --target wasm` gives native == wasm == 9.
# If a wasm gate ever goes red, pin it known-bad IN the wasm job (an XFAIL ledger, like
# test/native_fixtures/run), NOT by re-excluding it here — a red gate belongs in CI.
