# CI-COVERAGE-EXCEPTIONS.txt — gates deliberately NOT run in CI, each with a reason.
#
# diff_compiler_ci_shard_coverage.sh requires EVERY gate script to be either
#   (a) matched by a shard `pattern:` in .github/workflows/ci.yml, or
#   (b) named literally by some step in ANY workflow under .github/workflows/
#       (as of T-4 this includes nightly.yml, a scheduled home for gates
#       deliberately unsuited to the PR path — see the entry below), or
#   (c) listed HERE, with a reason.
#
# This file is a LEDGER, not a skip-list. The distinction is the whole point:
# a skip-list rots silently, because nothing ever revisits it. A ledger entry is
# PRINTED BY THE GATE ON EVERY RUN, names the reason, and names the task that
# retires it — so it is impossible to forget and expensive to leave lying around.
#
# Adding a line here is a deliberate, reviewable act. "It's broken so I excluded
# it" is only acceptable with a task number next to it and an owner fixing it.
#
# Format:  <gate-stem>  <reason>
