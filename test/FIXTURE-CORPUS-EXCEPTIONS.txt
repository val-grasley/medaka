# FIXTURE-CORPUS-EXCEPTIONS.txt — fixture/golden corpora deliberately NOT
# consumed by any gate, each with a reason.
#
# diff_compiler_fixture_corpus_coverage.sh requires every *fixtures*/*goldens*
# directory to be either (a) read by some gate (directly, or one hop through an
# invoked *.sh or *.mdk build/run target), or (b) listed HERE, with a reason.
#
# This file is a LEDGER, not a skip-list — same distinction as
# test/CI-COVERAGE-EXCEPTIONS.txt and test/CHECK-REMOVED-CONSTRUCTS-LEDGER.txt.
# A skip-list only checks "is this path excused" and stops looking, so it never
# notices when the excuse expires. This ledger is diffed in BOTH directions by
# the gate:
#
#   corpus has 0 consumers and is NOT here  -> FAIL (a real orphan)
#   corpus IS here but NOW HAS a consumer   -> FAIL ("this got wired up, delete
#                                                      the ledger line")
#
# The second direction is the one a skip-list structurally cannot see.
#
# Adding a line here is a deliberate, reviewable act. "Nothing reads it so I
# excluded it" is only acceptable with a real reason and, where the corpus is a
# genuine gap (not an intentional non-gate), an owner/task to close it.
#
# Format (TAB-separated):  <corpus-relpath>\t<reason>

test/bench_fixtures	only consumer is test/bench.sh, a timing/profiling harness that reports numbers and asserts nothing (see test/CI-COVERAGE-TOOLS.txt: "perf harness. Reports numbers; asserts nothing." — deliberately excluded from the gate-candidate set for the same reason preflight.sh treats it as non-gate infra). By design, not a gap: a benchmark corpus is not meant to be graded pass/fail. If a real regression gate ever starts reading these fixtures too, this line must be deleted (the bidirectional check will catch it).

test/doc_fixtures	real corpus (14 fixtures), zero consumers. `medaka doc` was ported to the native CLI (96b62efb, "feat(cli): port `medaka doc` to the native CLI (single-file)") but no `diff_compiler_doc.sh` golden-diff gate was ever built for it — the port shipped without regression coverage. Needs: a new test/diff_compiler_doc.sh capturing `medaka doc` output over these 14 fixtures as goldens (same shape as the other diff_compiler_* gates) plus enrollment in a CI shard. Not wired here deliberately — building it requires deciding the expected Markdown output per fixture, which is a real, reviewable piece of work, not a one-line fix. Owner: whoever picks up doc-tooling coverage.

test/index_fixtures	real corpus (2 fixtures: bare_bracket_index.mdk from #16c "add bare `a[i]` postfix indexing grammar", prelude_index_dispatch.mdk from d15c1b1e's Index-dispatch regression), zero consumers. Both fixtures carry a "Run with: medaka run test/index_fixtures/…" comment — written as manual-verification files and never wired into an automated gate. Not wired here deliberately — needs captured expected output first. Owner: whoever owns the #16c/Index-dispatch follow-up.

test/build_cmd_fixtures	real corpus (test/build_cmd.sh, a genuine golden-diff pass/fail gate: "Exit: 0 if every program builds and native stdout == golden") shows as UNCONSUMED only because test/CI-COVERAGE-TOOLS.txt (a DO-NOT-TOUCH file for this change) mislabels its stem `build_cmd` as "build helper" — the exact same TOOLS ledger diff_compiler_ci_shard_coverage.sh trusts to exclude non-gate infra, reused here for the same reason (see this gate's header). That mislabeling means test/build_cmd.sh is excluded from BOTH coverage gates' candidate sets, and it does not appear in any .github/workflows/*.yml pattern or named step either — so it likely runs in NO CI shard at all, hidden behind a bad TOOLS entry. This is a finding for whoever owns test/CI-COVERAGE-TOOLS.txt: `build_cmd` should almost certainly be removed from that ledger (it is a real gate, not a tool) and enrolled in a CI shard pattern (it fits the `backend` shard alongside `diff_compiler_build`). Once fixed there, this line self-resolves and the bidirectional check will demand its deletion.
