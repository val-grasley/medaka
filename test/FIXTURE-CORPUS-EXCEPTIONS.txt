# FIXTURE-CORPUS-EXCEPTIONS.txt — fixture/golden corpora deliberately NOT
# consumed by any gate, each with a reason.
#
# diff_compiler_fixture_corpus_coverage.sh requires every *fixtures*/*goldens*
# directory to be either (a) read by some gate (directly, or one hop through an
# invoked *.sh or *.mdk build/run target), or (b) listed HERE, with a reason.
#
# This file is a LEDGER, not a skip-list — same distinction as
# test/CI-COVERAGE-EXCEPTIONS.txt and test/CHECK-REMOVED-CONSTRUCTS-LEDGER.txt.
# A skip-list only checks "is this path excused" and stops looking, so it never
# notices when the excuse expires. This ledger is diffed in BOTH directions by
# the gate:
#
#   corpus has 0 consumers and is NOT here  -> FAIL (a real orphan)
#   corpus IS here but NOW HAS a consumer   -> FAIL ("this got wired up, delete
#                                                      the ledger line")
#
# The second direction is the one a skip-list structurally cannot see.
#
# Adding a line here is a deliberate, reviewable act. "Nothing reads it so I
# excluded it" is only acceptable with a real reason and, where the corpus is a
# genuine gap (not an intentional non-gate), an owner/task to close it.
#
# Format (TAB-separated):  <corpus-relpath>\t<reason>
