# test/EXTERN-DOMAIN-LEDGER.txt — per-extern DOMAIN classification (issue #476)
#
# WHY THIS EXISTS
#   diff_compiler_capability_matrix.sh proves each extern EXISTS on each engine.
#   diff_compiler_engines.sh proves the three engines AGREE on the sampled corpus.
#   Neither asks whether the engines agree at an extern's EDGE DOMAIN — the input
#   region where a partial function returns None / NaN / +-inf / saturates / traps.
#   floatToInt out of range was a three-way divergence (eval 0, native an ASLR
#   pointer — an S0 infoleak, wasm a trap) that survived for months with every
#   gate green, because NO fixture ever sampled the boundary (#346 / PR #473).
#
#   This ledger closes that hole STRUCTURALLY.  The gate DERIVES the set of
#   value-comparable externs (every extern in stdlib/runtime.mdk whose signature
#   carries NO capability annotation `<Cap>` — i.e. a pure, deterministic function
#   of its arguments, the only kind the value-differential can compare) and
#   REQUIRES each one to appear here exactly once with a domain verdict.  A new
#   pure extern with no row FAILS the gate — the classification cannot be
#   forgotten (self-draining, like CAPABILITY-EXCEPTIONS.txt).
#
# FORMAT  (tab-separated; '#'-prefixed and blank lines ignored)
#   extern <TAB> class <TAB> detail
#
#   class ∈ TOTAL | BOUNDARY | PENDING | EXEMPT
#
#   TOTAL    <reason>
#            Domain is total (or wraps/saturates by DESIGN): every input yields a
#            well-defined, finite, comparable value with no None/NaN/inf/OOB edge.
#            No boundary fixture required.  reason must say WHY it is total.
#
#   BOUNDARY <key[,key...]>  <reason>
#            Partial / edge-heavy domain WITH boundary fixtures pinning the edge
#            across all three engines.  Each <key> is a corpus/name pin key
#            (llvm/<n>, llvmT/<n>, wasm/<n>, wasmT/<n>).  The gate verifies, for
#            each key: the fixture .mdk exists, a matching .pin exists, and the
#            fixture SOURCE names this extern (so the row cannot point at an
#            unrelated cell).  The cross-engine value comparison itself is done by
#            diff_compiler_engines.sh, which auto-discovers the fixture.
#
#   PENDING  #<issue>  <reason>
#            Partial / edge-heavy domain whose boundary is NOT yet pinned.  A
#            LEGAL, GREEN, but COUNTED state: the gate reports the PENDING roster
#            every run, so the residual is visible and self-draining rather than
#            silently absent.  Converting a PENDING row to BOUNDARY (author the
#            cells) or TOTAL (prove it total) is the follow-up work #476 tracks.
#            detail MUST begin with an issue reference (#NNN).
#
#   EXEMPT   <reason>
#            Not a value-comparable function: aborting control flow, mutation,
#            IO plumbing, or a compiler-internal sentinel.  There is no value to
#            pin across engines.  reason must say which.
#
# HYGIENE the gate enforces (see diff_compiler_capability_matrix.sh §9):
#   - every DERIVED-pure extern has exactly one row here (missing => FAIL);
#   - no row names an unknown or non-pure (capability-annotated) extern;
#   - no duplicate extern rows;
#   - class is one of the four tokens;
#   - BOUNDARY keys resolve to real fixture+pin and the fixture names the extern;
#   - PENDING detail starts with #<issue>.
#
# ── BOUNDARY: edge-domain externs pinned across eval/native/wasm ──────────────
