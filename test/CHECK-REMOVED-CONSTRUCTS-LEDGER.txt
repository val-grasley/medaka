# CHECK-REMOVED-CONSTRUCTS-LEDGER.txt — expected findings for
# check_removed_constructs.sh, one per line.
#
# This is a LEDGER, not a skip-list. The distinction is the whole point (see
# test/CI-COVERAGE-EXCEPTIONS.txt for the canonical statement of it): a
# skip-list only checks "is this path excused" and stops looking, so it can
# never notice when the excuse expires — a deleted fixture or an accidentally
# -fixed diagnostic just silently vanishes from view. A ledger instead pins
# the EXACT SET of (file, construct) pairs the gate expects to find, and the
# gate diffs the ACTUAL finding set against this file in BOTH directions:
#
#   actual finding not in this ledger   -> FAIL ("new use of a removed
#                                           construct" — a real regression)
#   ledger entry not an actual finding  -> FAIL ("this got fixed or deleted —
#                                           remove the ledger entry below")
#
# The second direction is the one a skip-list structurally cannot see, and
# it's the reason this file exists instead of an in-script case statement.
#
# Every entry needs a real reason. "It's broken so I excluded it" is only
# acceptable with a task number next to it and an owner fixing it.
#
# Format (tab-separated):  <path>\t<construct>\t<reason>
# <construct> is the TIER1/TIER2 label check_removed_constructs.sh emits:
# let_mut, record_keyword, function_keyword, backtick_infix, default_impl,
# named_impl, let_else, at_name_hint.
