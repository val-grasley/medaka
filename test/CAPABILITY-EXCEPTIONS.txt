# test/CAPABILITY-EXCEPTIONS.txt — checked-in source of truth for every KNOWN
# (extern, engine) capability gap. Read test/CAPABILITY-MATRIX.md first.
#
# Consumed by test/diff_compiler_capability_matrix.sh. That gate FAILS if:
#   - an extern is unsupported by an engine and has NO row here (a "silent
#     omission" — exactly how the interpreter's 37-extern gap survived
#     undetected), or
#   - an extern DOES have a row here claiming it's unsupported, but the
#     engine's source now actually implements it (an "accidental fix" —
#     promote it: delete the row). Same principle as test/fuzz_allowlist.txt
#     in this arc (rustc's tests/crashes model).
#
# Format (TAB-separated; blank lines and '#' comments ignored):
#   extern<TAB>engine<TAB>category<TAB>reason[<TAB>pattern]
#
#   engine   ∈ interp | llvm | wasm
#   category ∈ BUG | DEAD | TODO | PERMANENT | TRAP-STUB | WASM-GAP | FROZEN-CONSTANT
#     BUG             — the extern works in >=1 OTHER engine; this engine's
#                        absence is a real regression, to be fixed (NOT a
#                        permanent exception — see PLAN.md for the tracked
#                        interpreter-Value-refactor workstream).
#     DEAD            — declared in runtime.mdk but not called by any current
#                        stdlib code path (superseded by a →MEDAKA rewrite);
#                        still directly reachable by a user program that
#                        names the extern explicitly, so not risk-free, but
#                        not an active regression either.
#     TODO            — unimplemented in EVERY engine; a planned/forward
#                        primitive with no caller yet, not an asymmetric gap.
#     PERMANENT       — will never be implemented by this engine for a
#                        documented structural reason (e.g. WasmGC has no
#                        raw-socket equivalent). Not expected to close.
#     TRAP-STUB       — bound in source, but to an `unreachable`/abort
#                        instruction rather than a real implementation; a
#                        program that actually evaluates it crashes loudly
#                        (better than FROZEN-CONSTANT's silent wrong value,
#                        but still not a real implementation).
#     WASM-GAP        — unported to the WasmGC backend; no structural
#                        blocker (unlike PERMANENT), just not done yet.
#     FROZEN-CONSTANT — WORSE than missing: the interpreter DOES bind this
#                        name (so `is_impl=Y` — no exceptions row is needed
#                        for the pass/fail check), but to a fabricated/no-op
#                        value instead of a real implementation, which is
#                        silent wrong behavior, not a loud panic. Tracked
#                        here anyway (per the task spec) so it can't be
#                        forgotten; the `pattern` column is a literal
#                        substring the gate greps for in eval.mdk on every
#                        run — if it disappears, the gate FAILS and asks a
#                        human to check whether it was fixed for real.
#
# Every row was hand-verified against source on 2026-07-13 (see the citations
# in the reason text and in diff_compiler_capability_matrix.sh's header).

# ============================================================================

# silently promoted to be the `medaka run` production engine and this gap was
# never re-litigated. Fix tracked as a separate, larger workstream (an
# effect-polymorphic Value refactor) — NOT in scope for this gate itself.
# ============================================================================
